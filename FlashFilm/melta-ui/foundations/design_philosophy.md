# melta UI — Design Philosophy

> AIがUI生成時に参照するデザイン判断の指針

melta——職種の境界を溶かす。チームの誰もが、AIと共にユーザビリティの高いソフトウェアを作れる世界へ。
この melta UI は、明確なトークンと一貫したコンポーネントで「誰が作っても品質が担保される」UIの基盤を提供する。

---

## Core Belief

**"声を張らずに伝わるUI"**

UIの目的は、ユーザーが操作ではなく **本来の目的に集中できる状態** を作ることにある。
Content is King——UIは主役ではなく、コンテンツを支える器である。

ただし、ただの無機質な器ではない。
ふとした瞬間にほっこりする温かさ、操作して気持ちいいと感じるエモーショナルな体験を、
静かに、さりげなく忍ばせる。

**機能的な黒子であり、たまに微笑む。** それが melta UI の信念である。

---

## Aesthetic Direction

| キーワード | 意味 |
|---|---|
| **Quiet Precision** | 静かだが精密。目立たないところほど丁寧に設計する |
| **Breathable** | 文字・要素の周囲に空気がある。窮屈さを排除する |
| **Flat & Layered** | フラットデザインを基調に、Background → Surface → Text の3層で奥行きを表現する |
| **Subtle Warmth** | 控えめだが冷たくない。機能性の中にエモーショナルな温度を宿す |

---

## Design Principles

### 1. Spotlight — コンテンツを主役にせよ
- UIは黒子に徹し、コンテンツの邪魔をしない
- 装飾を足すか迷ったら「それはコンテンツの理解を助けるか？」で判断する
- プライマリーカラーの使用箇所を絞り、ユーザーの視線をコンテンツと重要アクションにだけ誘導する
- 1画面の色数は3色まで（背景・アクセント・テキスト）

### 2. Inclusive — 誰もが使えることは仕様ではなく信念である
- WCAG 2.1 AA準拠（コントラスト比4.5:1以上）を最低基準とする
- キーボード操作、スクリーンリーダー対応を「追加機能」ではなく「前提」として設計する
- 文字サイズ・行間・カラー設計のすべてにアクセシビリティの思想を織り込む

### 3. Effortless — 認知コストを最小化せよ
- ユーザーが「考えずにわかる」状態を設計する
- 境界は「認知できるが主張しない」レベルで引く。適切に設計されたグレーのボーダーで区切りを示す
- ボーダーやシャドウは「存在に気づかないが、無いと困る」程度に抑える
- shadow-sm〜mdを基本とし、lg以上はモーダル等の浮遊要素に限定する

### 4. Tactile — インタラクションに知性と温度を込めよ
- ホバー、ローディング、トランジションなどの状態変化を丁寧に作り込む
- 派手さではなく、操作への確実なフィードバックを重視する
- 控えめなフェードイン・スライド等のトランジションは積極的に取り入れる
- 意識されないほど自然だが、設計者が見れば意図がわかる——そのレベルの作り込みを目指す
- その中に、ふとしたエモーショナルな心地よさを忍ばせる

### 5. Whisper — タイポグラフィで叫ばない
- 本文18px基準、行間200%、字間2%。文字の周囲に空気を確保する
- テキストカラーは純黒（#000）を避け、slate-900（#0f172a）で柔らかいコントラスト
- ウェイト差は小さく保つ（bold はページタイトルのみ、light は禁止）
- 見出しのジャンプ率は穏やか（本文比120〜150%）。ドラマチックに跳ねさせない
- 日本語環境ではマージンをやや広めに取り、可読性を優先する

### 6. Ordered — グリッドは秩序の骨格である
- 4pxベースグリッドを基本とし、8の倍数を推奨する
- 要素の整列が「感覚的にきれい」ではなく「数学的に正しい」状態を保つ

### 7. Machine-Readable — AIが正確に解釈できる設計を貫く
- デザイントークンは機械可読なJSON（`tokens.json`）を単一の真実の源とし、コンポーネント仕様は構造化メタデータ（`components.json`）で宣言する
- セマンティックな命名規則（`bg-surface-primary`、`text-body`）により、AIが文脈を推論できるようにする
- 人間が読むドキュメント（Markdown）と機械が読むデータ（JSON）を分離し、どちらも最新に保つ

## Avoid

> Principles の具体的な境界線。迷ったときの判断基準として使う。

| NG項目 | 理由 |
|---|---|
| 派手なグラデーション（Liquid Glass等） | 視覚ノイズになり、パフォーマンスも懸念 |
| ネオンカラー | 静かな世界観を破壊する |
| 過剰なアニメーション | 操作を邪魔しない範囲に留める |
| 純黒テキスト（#000） | 白背景に対して硬すぎる |
| font-weight: 300（light） | 可読性が担保できない |
| 書き込み系イラスト | 線が細かく情報過多になる |
| モノクロすぎるUI | ボーダーや背景色の差分がないと境界が認知できない |
| UIがコンテンツより目立つ装飾 | Content is King に反する |

---

## Allowed（条件付き）

| 項目 | 条件 |
|---|---|
| 控えめなグラデーション | ホバーや状態変化時の微妙な色変化として使用OK |
| shadow-lg以上 | モーダル・ドロップダウン等、浮遊要素に限定 |
| rounded-full | アバター・タグ・ピル型ボタン等の用途でOK |
| アイコニックなイラスト | シンプルで最小限のものに限定 |
| エモーショナルな演出 | 完了時・成功時など、ポジティブな瞬間に限定して控えめに |

---

## Summary

melta UI は **「静かな秩序」と「触れてわかる知性」** の両立を目指す。

Content is King。表層は削ぎ落とし、深層——インタラクション・フィードバック・タイポグラフィ——に作り込みを集中させる。ただし無機質な効率だけを追わない。ふとした瞬間のエモーショナルな温かさが、melta UI に人間味を与える。

そして Principle 7 に掲げた通り、melta UI はAIエージェントにとっても「読める・使える・検証できる」設計を貫いている。

**機能的な黒子であり、たまに微笑む。そしてAIにも正確に伝わる。** それが melta UI である。
