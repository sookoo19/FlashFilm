## フォントファミリー

```
font-sans: "Inter", "Hiragino Sans", "Hiragino Kaku Gothic ProN", "Noto Sans JP", sans-serif
font-mono: "JetBrains Mono", "SF Mono", monospace
```

Tailwind CSS 4のデフォルト`font-sans`を使用。日本語環境ではシステムフォントにフォールバック。

> プロジェクト固有のフォントは `foundations/theme.md` の Font Stack で上書き可能。

## 行間・字間の設計（デジタル庁の推奨値を参考に拡張）

CSSベースレイヤーで以下を適用:

```css
@layer base {
  body {
    line-height: 2.0;        /* 200% — 本文の可読性確保（デフォルト） */
    letter-spacing: 0.02em;  /* 2% — 日本語の自然な字間 */
  }
  h1, h2, h3, h4, h5, h6 {
    line-height: 1.4;        /* 140% — 見出しはコンパクトに */
    letter-spacing: 0.01em;  /* 1% — 見出しは控えめに */
  }
}
```

### ベースルール

- **body のデフォルト line-height は 2.0（200%）**: 18px 本文に対して行間を広くとることで、日本語の長文でもゆったりと読める。デジタル庁のデザインシステム推奨値（150%〜175%）を参考に、より余白を重視して独自に拡張
- **見出しの line-height は 1.4（140%）**: 見出しはコンパクトに保ち、本文との視覚的コントラストを明確にする
- **フォームコントロールのラベル包含要素には `leading-normal`（150%）を付与する**: body の `line-height: 2.0` はインライン `<label>` / `<span>` に継承される。親要素の line-height がラインボックスの最小高さを決めるため、`text-sm` 等で個別に line-height を指定しても親の 2.0 が勝つ。ラジオボタン・チェックボックス・トグルスイッチのラベル + 説明テキストを包含する `<div>` には `leading-normal` を付与してリセットする
- CSSベースレイヤーで自動適用されるため、上記のフォームコントロール例外を除き個別の line-height 指定は原則不要

**設計の背景**: 行間（200%）と字間（2%）を広げることで文字の周りに空気が生まれ、ゆったりと見える。本文を18pxに引き上げたことで、見出し全体もスケールアップし、情報の階層がより明確になった。字間（見出し1%・本文2%）と見出しの行間（140%）はデジタル庁デザインシステムの推奨値に沿い、本文の行間は推奨値（150%〜175%）を超えて200%に拡張している。

## ルール

- **本文のベースは `text-base`（18px）**: ゆったり路線。行間200%・字間2%と合わせて、長時間利用でも疲れない可読性を確保
- **純黒テキスト禁止**: `text-black` は使わず `text-slate-900`（#0f172a）を使う。青みがかった黒で長時間利用でも目に優しい
- **行間は用途で使い分ける**: 見出し140%、本文200%。CSSベースレイヤーで自動適用されるため、個別指定は原則不要
- **字間は微量に広げる**: `tracking-tight` は禁止。見出し1%、本文2%を基本とする
- **見出しと本文のサイズ比は120%〜150%**: テキストの階層をサイズで明確にする
