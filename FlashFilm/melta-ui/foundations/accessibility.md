# Accessibility

> WCAG 2.1 準拠のアクセシビリティルール。全コンポーネント共通。

---

## 信念

> "誰もが使えることは仕様ではなく信念である" — melta UI Design Principle #2

アクセシビリティは「追加機能」ではなく「前提」として設計する。
この信念を支える3つの柱:

1. **コントラスト** — テキスト・UI 要素の視認性を WCAG 4.5:1 以上で担保する
2. **キーボード操作** — すべてのインタラクティブ要素を Tab / Enter / Space / Escape で操作可能にする
3. **色＋テキスト併用** — 色だけで情報を伝達せず、必ずアイコン・テキストを併用する

---

## コントラスト要件

| 対象 | 最低比率 | WCAG基準 |
|------|---------|---------|
| 通常テキスト（16px未満） | 4.5:1 | AA |
| 大きなテキスト（18px以上 bold、24px以上） | 3:1 | AA |
| UIコンポーネント（アイコン、ボーダー） | 3:1 | AA |
| テキスト（AAA目標） | 7:1 | AAA |

---

## キーボードアクセシビリティ

### 必須事項

- すべてのインタラクティブ要素は **Tab キーでフォーカス可能** にする
- フォーカスインジケーターを **視覚的に明示** する（`focus:ring-2 focus:ring-primary-500/50`）
- フォーカス順序は **DOM順** に従う（`tabindex` の乱用禁止）
- Escape キーでモーダル/ドロップダウンを閉じられるようにする
- Space/Enter キーでボタンやチェックボックスを操作可能にする

### フォーカストラップ

モーダル表示中は:
- Tab/Shift+Tab がモーダル内を循環する
- モーダル外の要素にフォーカスが移動しない
- 閉じた後、トリガー要素にフォーカスが戻る

## ARIA属性

### 必須のARIA属性

| 要素 | 属性 | 用途 |
|------|------|------|
| モーダル | `role="dialog"` `aria-modal="true"` | ダイアログであることを示す |
| アラートモーダル | `role="alertdialog"` | 重要な通知であることを示す |
| モーダルタイトル | `aria-labelledby` | モーダルの目的を関連付ける |
| アイコンボタン | `aria-label` | テキストのない操作の説明 |
| エラーメッセージ | `aria-describedby` | 入力欄とエラーの関連付け |
| 無効状態 | `aria-disabled="true"` / `disabled` | 操作不可であることを示す |
| 不確定チェックボックス | `aria-checked="mixed"` | 一部選択状態を示す |
| バリデーションエラー | `aria-invalid="true"` | 入力値が不正であることを示す |
| トグル状態 | `aria-pressed="true/false"` | ON/OFF状態を示す |
| ライブ更新 | `aria-live="polite"` | 動的な状態変更を通知 |

## ラベルとテキスト

### 必須事項

- すべてのフォーム入力には `<label>` を関連付ける（`for` 属性 or ラッピング）
- チェックボックスグループには `<fieldset>` + `<legend>` を使用する
- プレースホルダーをラベルの代替にしない
- アイコンボタンには必ず `aria-label` を付与する
- 装飾以外のアイコンには代替テキストを付与する
- ボタンラベルは操作/遷移先を予測可能にする（「送信」「削除する」「詳細を見る」等）

---

## カラーアクセシビリティ

### 色だけで情報を伝達しない

| 状態 | 色のみ（禁止） | 正しい実装 |
|------|--------------|-----------|
| エラー | 赤いボーダーのみ | 赤ボーダー + エラーアイコン + テキスト |
| 選択状態 | 背景色のみ | 背景色 + ボーダー太さ変更 |
| 成功/失敗 | 緑/赤のバッジのみ | 色 + テキストラベル + アイコン |
| リンク | 色のみ | 色 + 下線 or アイコン |

---

## タップ/クリック領域

- インタラクティブ要素の最小タップ領域: **44 x 44px**
- ボタンの最小高さ: `h-8`（32px / Small）以上。デスクトップ標準は `h-10`（40px / Medium）、タッチ操作は `h-12`（48px / Large）
- アイコンボタン: `w-8 h-8`（Small）/ `w-10 h-10`（Medium）/ `w-12 h-12`（Large）の3サイズ
- インラインテキストリンクは例外（十分なパディングを確保すること）

---

## テキスト拡大

- テキストの200%拡大時にコンテンツがクリッピングしない
- レイアウトが破綻しない
- 3行以内に収まることを確認する

---

## アニメーション

- `transition-colors` で色変化のみ。duration はTailwindデフォルト（150ms）
- `transition-shadow hover:shadow-md` でホバー浮き上がり
- `transition-opacity duration-200` でモーダル出現
- **装飾的アニメーションは入れない**。状態変化の視覚フィードバックのみ
- `prefers-reduced-motion` を尊重する

---

## 禁止パターン（アクセシビリティ）

| 禁止 | 理由 | 代替 |
|------|------|------|
| ラベルなしのフォーム入力 | スクリーンリーダーが目的を読み上げられない | `<label>` を関連付ける |
| `aria-label` なしのアイコンボタン | 操作内容が不明 | `aria-label="閉じる"` 等を付与 |
| 色のみによる状態伝達 | 色覚多様性への非対応 | テキスト/アイコンを併用 |
| フォーカストラップなしのモーダル | キーボードユーザーが脱出できない | フォーカストラップを実装 |
| `outline: none` without ring | フォーカスインジケーターが消える | `focus:ring-2` を使用 |
| 最小サイズ未満のタップ領域 | 操作困難 | ボタンは `h-8` 以上（S: `h-8` / M: `h-10` / L: `h-12`） |
| `tabindex` > 0 | フォーカス順序が混乱する | DOM順に従う |
| 時間制限のある操作 | ユーザーが操作を完了できない | 時間制限を設けない |
| エラーメッセージの自動非表示 | ユーザーが読む前に消える | 明示的に閉じるまで表示 |
| プレースホルダーのみでラベル省略 | 入力開始で消え、目的がわからなくなる | `<label>` を必ず使用 |
