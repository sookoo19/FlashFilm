## これからすること

結論から言うと、成功しやすい進め方は
「AIにLUTそのものをいきなり学習させる」ではなく、AIに“その写真に対して最適な編集レシピを選ぶ/作る”ことを学習させる
です。

特にフラッシュ撮影限定なら、この方針はかなり現実的です。
ただし、設計を間違えると「学習はできたが使い物にならない」になりやすいので、順番が重要です。

1. まず前提を整理する
   LUTとは何か
   LUT は色変換表です。典型的には 3D LUT で、入力色
   (
   R
   ,
   G
   ,
   B
   )
   (R,G,B) を別の色へ写像します。

つまり LUT が得意なのは：

色の雰囲気を変える
全体のトーンを揃える
青み、赤み、コントラスト感、フィルム感を作る
一方で LUT だけでは苦手なものがあります。

白飛びしたハイライトの復元
顔だけを自然にする
肌色だけを守る
部分的なノイズ低減
局所的な明るさ補正
フラッシュ写真で本当に欲しくなりがちなのは、実はこの「LUTだけでは足りない部分」です。

なので、「自分の理想のフラッシュ写真」を作る仕組みは、厳密には
LUT + グローバル補正 + 顔/肌保護 + 場合によっては局所補正
です。

理由2: 制御しにくい

理由3: モバイル実装が難しくなる
今のアプリは ImageProcessingScreen.tsx で Aden、Clarendon、ColorMatrix を使う前提です。
ここから「写真ごとに 3D LUT を生成して GPU で適用する」へ行くと、レンダリング基盤から変える可能性が高いです。

4. 現実的な4段階ロードマップ
   段階A: 人手で“良いフラッシュルック”を定義する
   最初にやるべきは学習ではなく、美的基準の固定 です。

やること
フラッシュ写真だけを集める
自分が「これが良い」と思う仕上がりへ手で編集する
その編集を見て、共通パターンを言語化する
例えば：

肌の赤みを少し抑える
白飛び感を弱める
影を持ち上げすぎない
背景は暗く残す
フラッシュの硬さを少し和らげる
ここで重要なこと
「好きなLUTを学習」ではなく
“どういう見た目が好きか”を固定する
のが先です。

UIで手修正しやすい
データが少なくても始められる
実務では、この段階で十分「かなり賢い」体験になります。

11. 本当に 3D LUT へ行くなら、実装基盤も見直す
    今の react-native-image-filter-kit ベースは、固定フィルターやマトリクスには向いています。
    ただし、写真ごとに生成される 3D LUT を本格適用する汎用エンジン としては限界があります。

その場合の選択肢
サーバーで推論 + レンダリングして結果画像を返す
サーバーで推論して、アプリは LUT だけ受け取りネイティブで適用
iOS は Core Image / Metal ベースの独自ネイティブモジュール
将来的に Core ML / ONNX Runtime Mobile を使って on-device inference
今のスタック的な現実解
最初は
サーバーで AI 推論 → アプリには編集レシピだけ返す
が最も安定します。

理由は：

Expo/React Native で on-device 高速推論は重い
ネイティブ拡張の保守コストが高い
まずはモデル精度検証が先 12. よくある失敗

1. 編集者の好みがぶれる
   同じ写真でも日によって編集方針が変わると学習不能になります。
   最初は編集者を絞るべきです。

2. デバイスを混ぜすぎる
   端末ごとにフラッシュ色が違うので、最初は機種を絞った方が良いです。

3. 肌色の多様性が足りない
   特定の肌色だけで学習すると破綻します。これは非常に重要です。

4. LUTでできないものをLUTに押し込む
   肌補正、白飛び処理、ノイズ低減まで全部 LUT でやろうとすると無理が出ます。

5. 低信頼時の fallback がない
   AI は必ず外します。
   confidence が低いときは安全な既定レシピへ落とすべきです。

6. 実際に始めるなら、この順番が最も良い
   Phase 1: 2週間
   フラッシュ写真 300〜500枚収集
   自分の理想へ手編集
   編集方針を文章化
   Phase 2: 2〜4週間
   10〜20個のフラッシュ専用 LUT / レシピ作成
   各 before/after に対して最適レシピ探索
   AiEditRecipe データセット作成
   Phase 3: 2〜4週間
   lutId + strength + global params を予測する小型モデル学習
   まずはサーバー推論で良い
   Phase 4: 1〜2週間
   アプリに AIおすすめ を実装
   手修正ログを保存
   採用率を計測
   Phase 5: 継続改善
   顔 branch 追加
   肌保護ロス追加
   Basis LUT 合成へ拡張
   必要なら 3D LUT 生成へ進む
